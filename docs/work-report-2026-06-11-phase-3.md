# InnerOS Phase 3 阶段工作报告

日期：2026-06-11

## 产品判断

本阶段继续围绕同一个发心：用户只负责留下真实记录，系统自动把记录转化为可追溯的自我认识，并在今天需要判断时主动浮现。

因此没有继续增加彼此孤立的 AI 功能，而是收口为两条主链路：

```text
记录 -> 主题/记忆 -> 洞察 -> 准则
记录 -> 今日问题/行动 -> 对话 -> 新记录
```

## 已完成

### 统一对话

- 移除“回溯/行动”模式切换。
- 新增统一 Prompt，由模型判断当前应回看、澄清、推进或混合回答。
- 历史会话统一迁移为 `unified`。
- 不再因行动模式额外拼入大量近期笔记。

### 记忆 v4 与 Token 控制

- 历史 `memory.link` 基线：平均输入 5,286 Token/次，总输入约 708 万 Token。
- 候选记忆由 30 条降至 12 条。
- 原文上限由 5,000 字符降至 2,600 字符。
- 候选证据缩为每条 2 段、每段 160 字符。
- 纯摘录、资料和无长期信号的极短笔记由本地规则跳过。
- 单次记忆输出由最多 6 条降至 4 条。
- 新增信念、模式、偏好、约束四类记忆。
- 清理 2 条无原始证据的孤立记忆。

### 认识自己

- 侧边栏“主题/洞察”合并为“认识”。
- 新页面统一展示主题、记忆、洞察、准则。
- 记忆支持类型筛选、搜索、置信度、证据数量和证据浮窗。
- 主题详情仍保留独立深挖页面。

### 今日首页

- 一次 `today.digest` 调用同时生成 3 个问题和 3 个行动。
- 问题与行动分别自动轮播，也可手动切换。
- 实测平均输入约 3,388 Token/次，低于旧版只生成一个行动的约 5,114 Token/次。
- 首页入口直接带着具体问题或行动进入统一对话。

### 引用

- LLM 行内引用由日期改为笔记标题。
- “相关记录”卡片以标题为主，日期降为辅助信息。
- 点击标题引用仍打开原始笔记浮窗。

## 尚未强行处理

存量记忆中仍有 469 条 `event`。旧 Prompt 曾把观点、偏好和单次经历大量归入事件。

本阶段没有用关键词直接批量改写这些历史语义，因为这种做法成本低但误伤风险高。下一步应做可回滚的分批重整：每批只处理少量高相似事件，先合并重复，再重分类为信念、模式、偏好或约束，并记录变更依据与 Token 成本。
