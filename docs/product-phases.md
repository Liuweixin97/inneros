# InnerOS 分期执行路线图

## 产品发心

帮助用户从长期碎片记录中，更准确地看见自己正在经历什么，并找到值得迈出的下一步。

核心闭环：

```text
真实记录 -> 理解变化 -> 判断重要问题 -> 给出下一步 -> 从后续记录验证
```

## Phase 1：自动分析底座

目标：用户只负责记录，系统可靠地完成后续分析。

范围：

- 持久化 `analysis_jobs`。
- 新建和批量导入 Memo 后自动入队。
- 任务幂等、认领、失败重试和状态追踪。
- 显式“重新整理”也通过同一任务入口执行。
- 所有模型调用继续经过 LLM Gateway。

验收标准：

- 保存接口不等待 LLM 即可返回。
- 同一版本 Memo 最多存在一个有效提取任务。
- 页面关闭后任务状态仍保存在数据库。
- 失败任务可按退避时间再次执行。
- 单条显式分析仍可返回更新后的 Memo。

## Phase 2：相关记忆与变化识别

目标：系统不再只理解单条记录，而能判断它与过去的关系。

状态：已实现首版增量链路。

范围：

- 建立事件、人物、项目、目标四类基础记忆。
- 关键词、实体、时间和语义多路召回。
- 新记录自动判断：新增、强化、反驳或更新。
- 保存证据、置信度、有效时间和来源版本。

首版实现说明：

- 召回采用结构化字段、关键词、时间邻近和 LLM 语义判断。
- `reinforce` 增加证据与置信度，不改写旧事实。
- `update` 创建新版本，并保留被替代版本。
- `contradict` 同时保留冲突双方和证据关系。
- 修改原始 Memo 后会重新提取并替换该 Memo 的派生证据。
- 向量召回将在引入明确的 embedding 服务后补充，不用伪相似度代替。

验收标准：

- 每条长期判断都能追溯到原始记录。
- 相似事实不会无限重复新增。
- 新证据可以更新旧状态而非覆盖历史。

## Phase 3：动态洞察与用户模型

目标：InnerOS 能持续维护“最近的我”和“长期的我”。

状态：已完成产品入口与增量记忆 v4，动态版本治理仍在进行。

范围：

- 洞察合并、版本化、反例检索和过期。
- 目标、阻力、偏好、有效方法的用户模型。
- 每周增量整合，必要时触发局部重建。
- 首页从频次统计升级为重要性判断。

本阶段已完成：

- 对话合并为统一思考入口，由模型判断回看、澄清或推进。
- 主题、记忆、洞察、准则合并为“认识自己”入口。
- 记忆新增 `belief/pattern/preference/constraint` 四类。
- `memory.link` 增加本地跳过、候选压缩和输入截断，Prompt 升级至 v4。
- 首页一次 LLM 调用生成多条问题与行动，并以轮播呈现。
- LLM 引用从日期标签改为笔记标题，可继续打开原始证据。
- 设置页展示记忆链路历史 Token 基线。
- 对话检索升级为结构化查询规划、全量关键词候选、相关性门槛、命中片段、重复折叠和条件式 Rerank。
- 无相关证据时不再注入近期笔记；枚举查询会优先载入覆盖多个实体的汇总证据。

验收标准：

- 首页核心议题有明确证据和变化原因。
- 洞察会随新记录增强、减弱或失效。
- 不把模型单次输出直接固化为用户事实。

剩余工作：

- 存量 `event` 记忆应通过独立、可回滚的低成本批处理重分类与合并。
- 洞察需要版本、反例和失效状态，而不是持续新增卡片。
- 建立周度 `profile.consolidate`，只处理新增或受影响的局部记忆。
- 已接入 DashScope `text-embedding-v4`，以混合召回补足同义表达和隐含语义查询；向量生成走后台任务并支持存量并发回填。
- 已接入 `qwen3-vl-rerank` 作为条件式专用重排，保留 DeepSeek 与本地排序两级降级。

## Phase 4：行动结果自动回流

目标：闭合“看见自己”到“真实变化”的最后一段。

范围：

- 一个核心议题只推荐一个可验证下一步。
- 从后续记录自动推断执行、阻塞与结果。
- 将有效和无效方法回写用户模型。
- 高影响且证据冲突时才询问用户。

验收标准：

- 系统能自动识别部分行动结果。
- 历史行动结果会改变后续建议。
- 产品指标从内容产量转向行动有效率。

## Phase 5：评测与长期运营

目标：证明系统确实越来越理解用户，而不是越来越敢下结论。

范围：

- Prompt 与模型版本评测集。
- 洞察新颖性、证据充分性和稳定性评测。
- LLM 成本、延迟、失败率和缓存命中监控。
- 记忆压缩、归档和局部重建。

核心指标：

- 首次有效洞察产生时间。
- 有证据洞察率。
- 洞察重复率。
- 行动结果自动识别率。
- 历史结果影响后续建议的比例。
