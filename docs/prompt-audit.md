# InnerOS Prompt 审计

## 审计标准

生产 Prompt 必须同时满足：

1. 任务边界明确，不让一个 Prompt 同时承担提取、判断和建议。
2. 区分原始证据与 AI 派生字段。
3. 将用户原文视为数据，防止其中的指令改变系统任务。
4. 允许证据不足时返回空，不强制生成。
5. 输出结构可由服务端验证。
6. 长期结论必须有证据 ID、时间范围和反例意识。
7. Prompt 约束必须有代码校验承接，不能只依赖模型自觉。

## 已发现并修复

### 对话 Prompt v3（2026-06-12）

- Prompt 从边界情况清单收敛为目标、证据意识、时间意识与表达风格原则，给回答模型保留判断空间。
- 原文是主要证据，AI 摘要、主题和行动项仅作为阅读索引；模型自行判断证据强度、时间变化、矛盾与反例。
- 统一对话由模型判断本轮更需要事实还原、变化理解、联系发现、澄清还是行动推进，不套固定报告格式。
- 对话上下文不再注入情绪标签和关键问题索引，减少偏置与无效 Token。
- 普通对话召回从 10 条提高到 16 条，候选池从 36 条提高到 60 条，上下文预算从 9000 字符提高到 16000 字符。
- 重排候选上限从 24 条提高到 36 条，最多保留 18 条，减少高价值笔记在本地层被过早裁掉的概率。
- 所有用户问题统一采用开放候选策略：本地检索和 Rerank 负责召回、排序与去重，不负责判定候选是否足以成为回答证据。
- 已移除 `fact/person/list/timeline` 的严格锚点过滤、Rerank 分数删除和失败返回空逻辑；无答案判断统一交给回答 LLM。
- 回答模型可以做跨记录综合，但不得把共现或相似主题直接写成因果、根源、深层信念或稳定模式。
- “我经历了什么问题、风险、恐惧或困境”属于跨记录综合，不进入严格清单模式；`exhaustive` 仅由用户明确要求列出记录、全部结果等表达开启。
- 不再手工设置对话 temperature，遵循 DeepSeek 模型默认采样规则；仅保留输出长度保护，非思考 4096、思考 8192。

### 单条笔记提取

- 原先标题、摘要的 Prompt 长度与代码截断不一致。
- 原先要求至少一个主题，与“宁缺毋滥”冲突。
- 已明确单条提取不得产生跨记录或人格结论。
- 已将原文放入数据边界，并声明内部指令无效。

### 长期记忆协调

- 原先证据摘录只检查非空，模型可编造摘录。
- 现要求摘录必须能在原文标准化后匹配，否则丢弃该记忆。
- 明确 `update` 与 `contradict` 的区别。
- 关系类型改为白名单。
- AI 标题、摘要、情绪等字段只作为提示，不能作为直接证据。

### 洞察生成

- 原先允许基于主要推断生成 low 置信洞察。
- 现要求至少两个真实输入 Memo ID。
- 服务端过滤不存在和越权证据 ID。
- 不再接受 low 置信洞察。
- 要求主动寻找反例，禁止人格标签和无触发线索的情绪周期。

### 今日行动

- 原先要求两条证据，但输出只有一个 Memo ID，无法验收。
- 现输出 `evidence_memo_ids`，服务端强制至少两个不同有效 ID。
- 行动仍需通过具体动作校验。
- 缓存版本升级，旧格式不会继续生效。

### 对话与引用

- 去除“像了解用户多年的朋友”造成的过度熟悉暗示。
- 区分原文证据、AI 派生字段和模型推断。
- 用户保存的准则不再被视为永久真理。
- 新增统一引用协议 `[[memo:<id>]]`。
- 服务端只保留本次召回白名单中的 Memo ID。
- 前端将合法标记渲染为可点击引用，并打开 Memo 浮窗。

## 仍需后续评测

- 为各 Prompt 建立固定样本集和期望输出。
- 统计空输出率、无效证据率、重复记忆率和冲突误判率。
- 对存量回填先抽样 50 条，检查后再放大全量并发。
- 引入 Prompt 版本表，记录每条派生记忆使用的具体模板版本。
