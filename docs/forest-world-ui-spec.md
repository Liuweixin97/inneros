# 林间世界 UI 规范

本文档定义林间世界后续所有场景的统一 UI 语言。它不是营销页规范，也不是普通 SaaS 面板规范，而是一个「像素地图上的纸张式交互层」。

## 产品定位

林间世界的目标是帮助用户回到自己的记录里，看见最近的自己。它不以任务打卡、成长建议或 AI 输出为中心。

每个场景只承担一个清晰闭环：

- **记忆花园**：让原始记忆现身，并允许用户带走一段。
- **亮灯木屋**：给出今日路线，承接离开/设置/回到 InnerOS。
- **门前长椅**：决定同行方式和权限边界。
- **篝火地**：围绕一段材料，把此刻的说法放清楚。
- **静水池塘**：不调用 AI，负责暂停、释放、移出行囊。
- **循光小径**：确认多段记忆是否真的有关，并命名关系。
- **共居工坊**：让过去的我和现在的我并置，不强行合并。
- **中庭写作台**：把本次旅程确认的一句话写回真实 Memo。

## 视觉原则

### 1. 地图是世界，UI 是纸

地图、人物、物件资产保持原样。所有交互层应像临时摊开的纸、木牌、账本、便笺，而不是现代软件弹窗。

### 2. 小圆角，少装饰

卡片和面板圆角控制在 **3-4px**。避免大圆角、玻璃拟态、渐变光球、浮夸阴影和装饰性胶囊。

### 3. 纸张优先，深色只做遮罩

主面板使用浅纸色，正文使用墨色。深色森林只作为背景和遮罩，不作为主要阅读面。

### 4. 列表行优先，不堆大卡片

记忆选择、材料选择、来源证据都使用紧凑列表行。只有场景主动作、空状态、对照板可以用较大的纸块。

### 5. 不用 UI 替用户下结论

按钮文案应表达动作，而不是评价用户。例如使用「放在一起」「分开放回去」「留下这句话」，避免「这是正确答案」「你应该」。

## 面板规范

### 桌面端

- 面板宽度使用 `min()` 限制，不横跨全屏。
- 面板最大高度为 `calc(100dvh - 56px)`。
- 内容超出时只允许面板内部滚动。
- 背景地图必须可见，但通过深色遮罩降低干扰。
- 关闭按钮固定在右上角，尺寸 34px。

### 移动端

- 场景面板和桌面端一样在当前视口居中，不使用底部抽屉。
- 宽度 `calc(100vw - 24px)`。
- 最大高度 `min(86dvh, calc(100dvh - 24px))`。
- footer 在底部，允许换行，不覆盖正文。
- 操作按钮最小高度 38px。
- 多步骤进度条使用 2 列网格；奇数最后一步跨整行，避免截断或半露出。

## 组件规范

### Header

每个场景面板必须有：

- `kicker`：地点 + 场景定位，例如「篝火地 · 放清楚」。
- `title`：本场景用户要完成的核心动作。
- `subtitle`：解释边界，不写功能说明书。

### Button

按钮分四类：

- `primary`：当前闭环的下一步。
- `secondary`：同级替代动作。
- `quiet`：关闭、返回、不做。
- `danger`：删除、撤回、清空。

按钮不使用胶囊形。文本可配 lucide 图标，但图标必须服务动作。

### Progress

多步骤场景必须显示步骤条。步骤条表达“现在在哪一步”，不是完成率炫耀。

### Memo Row

记忆选择使用统一行组件：

- 左侧标题
- 下方日期 + 摘要
- 右侧选中状态或动作
- 行高稳定，避免 hover 或选中导致布局跳动

### Empty State

空状态必须说明用户现在能做什么，而不是只说「暂无数据」。

## 场景闭环规范

### 亮灯木屋

输入：近期 Memo、旅程事件。  
动作：查看今日路线、打开设置、返回 InnerOS。  
输出：无新 AI、无新数据，负责导航和收束。

### 门前长椅

输入：当前 companion mode。  
动作：选择一个人走、邀请苔灯、同屏双人。  
输出：同行方式变化。必须明确权限边界。

### 篝火地

输入：最多三段行囊记忆。  
动作：选择谈法、选择材料、写一句话或向苔灯发起对话。  
输出：旅程事件 `fireside_note` 或 `left_question`。  
约束：AI 只看用户本次选择的材料。

### 静水池塘

输入：行囊、旅程事件、本机漂流瓶。  
动作：坐一会儿、写给三天后的自己、从行囊移出。  
输出：`pond_release` 或本机漂流瓶。  
约束：不调用 AI。

### 循光小径

输入：两到三段 Memo。  
动作：选择纸条、判断是否同路、命名或拆开。  
输出：`named_path` 或 `separated_path`。  
约束：苔灯只提出候选，用户确认关系。

### 共居工坊

输入：一段旧记录。  
动作：选择问题、写现在回应、选择保存方式。  
输出：世界物件 + `placed_object`。  
约束：不合成正确答案，只保存并置关系。

### 中庭写作台

输入：行囊、旅程事件。  
动作：写成一条新的 Memo。  
输出：普通 Memo，进入正常分析队列。  
约束：这是唯一把旅程写回 InnerOS 的地方。

## 验收标准

一个场景完成必须同时满足：

- 使用统一 `ForestSceneLayer` / `ForestScenePanel` / `SceneButton` / `SceneMemoCard`。
- 视觉上是纸张式面板，非现代大圆角浮层。
- 所有浮窗在桌面端和移动端都必须相对当前视口居中。
- 桌面端不溢出、不遮挡 HUD 关键控制。
- 移动端不横向滚动，footer 不盖正文。
- 文案只表达用户动作和产品边界，不替用户下结论。
- 场景有明确输入、动作、输出。
- AI 相关场景必须说明授权边界。
