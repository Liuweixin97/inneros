# InnerOS 开源产品介绍

InnerOS 是一个面向个人记忆、自我观察和长期反思的开源应用。它把日常笔记、摘录、情绪、对话和 AI 生成的分析分层保存，并通过记录系统、时间线、主题洞察和可漫游的像素风「林间世界」帮助用户重新看见自己的生活线索。

InnerOS 的发心不是让 AI 替用户给建议，也不是把人生经验加工成标准答案。它更像一个安静的个人记忆操作系统：保存真实记录，整理可追踪的证据，在合适的时候提出问题，让用户自己判断什么重要。

## 为什么做 InnerOS

传统笔记工具擅长存储，却常常让内容沉没；很多 AI 工具擅长回答，却容易把复杂经验压缩成一句建议。InnerOS 试图在二者之间建立新的产品形态：

- 记录仍然属于用户。
- AI 分析只是辅助层，不覆盖原文。
- 每个洞察都应该能回到证据。
- 回看记录不应该只有列表，也可以是一个空间。
- AI 应该降低整理成本，而不是制造更多焦虑。

## 核心体验

### 1. 记录真实发生的内容

用户可以快速写下想法、事件、摘录、情绪、任务或任意片段。系统会保留原始表达，并在此基础上生成标题、情绪标签、主题线索和可检索的结构化信息。

### 2. 把 AI 作为整理层

InnerOS 使用大语言模型进行标题生成、情绪识别、主题归纳、洞察沉淀和对话陪伴。AI 的输出会进入数据库缓存和归档，避免页面每次刷新都重复请求模型，也避免不必要的 token 消耗。

### 3. 通过时间线和主题重新遇见自己

时间线帮助用户看到变化，主题帮助用户看到反复出现的问题。认识页则沉淀更稳定的观察，并保留对应证据，避免 AI 输出变成不可追踪的结论。

### 4. 在林间世界里回看记忆

林间世界是 InnerOS 的空间化记忆界面。用户可以在地图中移动，靠近记忆花园、篝火长椅、静水池塘、循光寻迹、共居工坊等区域，以更柔和的方式重新打开自己的记录。

## 主要模块

- **今日记录**：快速记录当天的想法和片段。
- **记录列表**：查看、筛选、编辑和回看历史内容。
- **主题页**：聚合同类问题和反复出现的线索。
- **时间线**：按时间顺序理解自己的状态变化。
- **认识页**：保存 AI 辅助生成的洞察和证据。
- **AI 对话**：基于用户授权的记录进行上下文回应。
- **多账户系统**：支持账户名注册、登录、退出、密码重设和个人资料维护。
- **游客模式**：允许未登录用户体验核心产品路径。
- **林间世界**：把记录映射为可漫游、可阅读、可回应的空间。

## 林间世界

林间世界不是任务驱动游戏，而是一个低压力的记忆回看空间。

- **亮灯木屋**：进入世界和返回主应用的入口。
- **记忆花园**：把 Memo 映射成风铃、小灯、信件和植物。
- **篝火长椅**：与 AI 同行者「苔灯」进行倾听、追问、整理或静默陪伴。
- **静水池塘**：写下不想被分析的内容，并把它作为漂流瓶放下。
- **循光寻迹**：从记忆碎片中找到线索，逐步点亮路径。
- **共居工坊**：围绕共同记忆进行共写和确认。
- **中间写作台**：连接地图漫游与正式记录动作。

## 技术实现

InnerOS 使用 Next.js、React、TypeScript、SQLite、Canvas 和 GitHub Actions 构建。AI 链路支持大语言模型、embedding、rerank、RAG 检索、后台分析任务和结果缓存。项目可以本地运行，也可以通过 Docker 自部署。

技术栈包括：

- Next.js 16
- React 19
- TypeScript
- Tailwind CSS v4
- SQLite / better-sqlite3
- Zustand
- Canvas
- DeepSeek API
- DashScope Embedding / Rerank
- Docker / Docker Compose
- GitHub Actions CI/CD

## 开源方向

InnerOS 当前是一个早期但可运行的产品原型。开源后，项目希望吸引对以下方向感兴趣的开发者和设计者：

- 个人知识管理
- AI 记忆系统
- 本地优先应用
- 情绪与自我观察产品
- 游戏化但不任务化的交互
- 可解释、可追踪的 AI 分析
- 自部署和隐私友好的 AI 工具

## 项目原则

1. 原始记录永远优先于 AI 解释。
2. AI 只做整理和陪伴，不替用户下结论。
3. 所有洞察都应该能追溯到证据。
4. 用户需要能理解自己的数据在哪里。
5. 产品应该帮助用户看见自己，而不是评价自己。
