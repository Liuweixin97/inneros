# InnerOS Agent SOP

## 基本原则
- 新消息优先。用户中断或改口后，立即停止旧任务，不继续执行旧计划。
- 先确认事实，再下结论。不能只看代码或记忆判断“已经改了”，必须确认当前分支、运行页面、API、构建和浏览器状态。
- 不把旧版 UI 当成新版成果。发现浏览器显示和目标方案不一致时，先承认偏差并定位分支/缓存/构建/实现原因。
- 不覆盖用户未要求回退的改动。任何切分支、保存待定分支、清理文件前，先看 `git status`。
- 产品和工程同等重要。对模糊方案要先做 PM/PD 评审，再落地，不机械照抄旧实现。

## 每次操作 SOP
1. **确认任务边界**
   - 复述最新目标和停止条件。
   - 明确当前仓库、分支、线上/本地 URL、是否要部署。
   - 如果用户要求“停止”，只做其明确要求的收尾动作。

2. **检查现场**
   - 运行 `git branch --show-current` 和 `git status --short`。
   - 读取相关代码、配置、API、数据库 schema，不凭记忆判断。
   - 对 UI 任务，必须打开实际页面确认当前显示版本。

3. **产品评审**
   - 先问：这个功能是否服务 InnerOS 的核心目标：帮助用户看见最近的自己，而不是给用户制造任务压力或替用户下结论。
   - 删除或打回职责不清、重复、多人化、奖励化、过度游戏化的功能。
   - 场景、命名、文案、交互要围绕用户实际路径闭环，而不是围绕已有代码结构。

4. **实现**
   - 优先沿用项目现有技术栈和风格。
   - 改动要覆盖 UI、API、类型、数据库、文案、测试和部署配置的完整链路。
   - 不保留“旧入口可用、新入口也可用”的多版本混乱状态，除非用户明确要求冻结分支。

5. **验收**
   - 至少运行 `npm run lint` 和 `npm run build`。
   - API 改动必须用 `curl` 或等价方式做 smoke test。
   - UI 改动必须用浏览器检查真实渲染、点击、弹窗位置、桌面和移动端关键状态。
   - 不能只看到 SSR HTML 就说交互正常；必须确认 hydration、控制台、按钮点击后的状态变化。
   - 若使用 `127.0.0.1` 调试 Next dev，确认 `allowedDevOrigins` 已包含该 origin，否则客户端脚本可能被拦。

6. **交付**
   - 最终回复只说已完成的事实、改了哪些关键点、跑过哪些验证、还剩什么风险。
   - 未跑通的测试必须明确说清。
   - 不夸大，不把“代码看起来有”说成“产品已经可用”。

## InnerOS 特殊要求
- 登录只使用账户名和密码，不引入邮箱登录。
- 不提供游客访问；主程序、林间世界和业务 API 都必须在注册或登录后使用。
- 多用户数据必须按真实用户隔离。
- AI 输出要尽量缓存和归档，避免重复调用消耗 token；用户确认过的结果才进入长期回看。
- AI 不能替用户下人生建议，只能帮助用户看见关系、重复主题、情绪线索和问题线索。

## 林间世界产品要求
- 外层仍可叫“林间世界”，内部是单人低压力回看路径。
- 保留核心地图和人物资产，但允许大幅重构 UI、场景、命名、API 和数据模型。
- 当前目标循环是：`入林长椅 -> 记忆花园 -> 观照桌 -> 苔灯火边 / 静水池 -> 亮灯木屋`。
- 删除多人、共写、同屏双人、Player 1/2、双方确认、多人大纲、多人大 draft API。
- 不使用 Day、今日任务、配方、酿造、合成、奖励、道具栏式底栏等压力型游戏语言。
- “本次带入”最多 3 段记忆，是临时回看上下文，不是背包养成系统。

## 6 个场景职责
- **入林长椅**：只选择“独自走走”或“请苔灯同行”，不承载复杂功能。
- **记忆花园**：原始记录入口；打开最近笔记、加入本次带入、留一句今天的话；不做任务和奖励。
- **观照桌**：单人关系观察核心；把 1-3 段记忆放在一起，看重复主题、情绪、问题线索，并保存“我的看法”。
- **苔灯火边**：AI 倾听和追问区；必须显示“仅基于你本次带入的记忆”，且只读取用户授权的 1-3 段记忆。
- **静水池**：反 AI、反分析区；可以放下、沉默、写临时漂流瓶；不调用 AI，不长期归档。
- **亮灯木屋**：回看终点；只展示用户亲自确认保存过的木牌、问题、火边记录、观照结果和周回看。

## 林间世界验收清单
- `rg "human_local|co_write|cowrite|Player 2|双人|共写|共居工坊|SharedMemoryDraft|workshop|LightTrail|light-trail|配方|酿造|合成|今日任务|Day 13|道具栏|同屏"` 在源码关键目录中为 0。
- 6 个地图点都能进入，浮窗居中，桌面和移动端都不遮挡主路径。
- 从记忆花园选择记忆，到观照桌保存看法，再到亮灯木屋回看，形成闭环。
- 静水池不触发 AI。
- 苔灯火边无授权记忆时不能发起对话。
- 设置界面没有多人/同屏/同行方式，只保留外观、文字、动态和操作说明。

## UI 偏好
- 偏好纸页式、直角或小圆角、有棱角的 UI，不喜欢大弧形和泛泛的卡片堆叠。
- 林间世界 UI 要有星露谷式温暖感，但不能变成任务压力游戏。
- 所有浮窗默认居中；移动端不能出现文字遮挡、按钮溢出、面板盖住关键路径。
- 视觉修改不能只靠截图想象，必须实际打开浏览器检查。

## 基础设施要求
- Flowork 和 InnerOS 的 GitHub Actions、服务器、DNS、HTTPS 配置必须指向当前新服务器。
- 修改 CI/CD 前先读现有 workflow、服务器目录、服务管理方式和 secrets 需求。
- SSH 操作前先确认目标主机、用户、密钥和风险；避免不可逆操作。
- 部署后必须检查服务状态、域名解析、HTTPS、关键 API 和页面可访问性。
